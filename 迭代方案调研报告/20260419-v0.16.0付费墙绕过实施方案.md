# P0 迭代实施方案 — 付费墙绕过 + JSON-LD articleBody 提取

- **目标版本**：feedgrab v0.16.0
- **规划日期**：2026-04-19
- **参考调研**：`迭代方案调研报告/20260419-qiaomu-anything-to-notebooklm对比分析.md`

## Context

feedgrab 当前对未识别 URL（generic 路径）**只用 Jina Reader 一层兜底**，对 300+ 主流付费新闻网站（NYT/WSJ/FT/Economist/Bloomberg/SCMP/Medium 等）几乎完全无感 —— Jina 能抓到部分软付费墙，但对硬付费墙（The Information、Bloomberg premium）基本返回付费墙提示页而非正文。

`qiaomu-anything-to-notebooklm` 的 `scripts/fetch_url.sh`（380 行 Bash）核心贡献是**付费墙 6 层级联绕过 + JSON-LD articleBody 提取**，经过用户同意后，本次将该能力**用 Python 重写**融合进 feedgrab，作为 Jina 之前的前置兜底层。预期成果：**对 300+ 付费新闻网站实现"首页 URL → 正文 Markdown"的开箱能力**，且对非付费站点的 JSON-LD 提取也提供一条轻量快速路径（比 Jina 快 5-10 倍）。

### paywall.py Tier 分级（7 级级联）

| Tier | 策略 | 触发域名 | 耗时预估 |
|------|------|---------|---------|
| 1 | Googlebot UA + `X-Forwarded-For: 66.249.66.1` + Google Referer + **JSON-LD 优先** | `GOOGLEBOT_DOMAINS` (~22 站) | <2s |
| 2 | Bingbot UA + Bing Referer + JSON-LD 优先 | `BINGBOT_DOMAINS` (4 站) | <2s |
| 3 | 通用策略对 `PAYWALL_DOMAINS` (44 站)：Googlebot → Bingbot → FB Referer → Twitter Referer | 所有付费墙 | 2-8s |
| 4 | AMP 页面重定向（尝试 4 种 AMP URL 模式） | `AMP_DOMAINS` (~12 站) | 2-4s |
| 5 | EU IP (`X-Forwarded-For: 185.X.X.X`) + Google Referer | 所有付费墙 | <2s |
| 6 | archive.today（CAPTCHA 自动检测，命中则 logger.warning + 返回 None 让 Jina 接手） | 全部 | 3-8s |
| 7 | Google Cache | 全部 | 3-5s |

## 关键设计决策

### 决策 1：新增 `SourceType.WEB` 而非复用 `MANUAL`
理由：让付费墙抓的新闻落到 `output/Web/` 独立目录，和其他未分类内容（`MANUAL` → `Other`）隔离。用户可直接在 Obsidian 里按 Web/Article 视图索引。改动成本低（2 行 + 1 个 from_web）。

### 决策 2：JSON-LD 提取独立成 `utils/jsonld.py`
理由：非付费站点也常带 JSON-LD（SEO 标配）。未来可能给 GitHub README / Notion / Medium free 文章复用。独立出来避免耦合。

### 决策 3：不移植 `scripts/fetch_url.sh` 的 Bash 实现
理由：(1) Windows 无 Git Bash 用户不能用；(2) feedgrab 所有 fetcher 都是 Python，保持一致；(3) Python 有 `utils/http_client.py` 的 curl_cffi Chrome TLS 指纹，比原始 curl 更难被检测。

### 决策 4：archive.today CAPTCHA 检测降级为"跳过 + 日志"，不是"exit 75"
理由：feedgrab 是库，不是 Bash 工具，不能 exit。检测到 CAPTCHA 时 `logger.warning` 提示用户 "archive.ph 需人机验证，已跳过"，让流程继续走 Google Cache / Jina。用户看日志后可手动打开 archive URL 验证。

### 决策 5：域名列表硬编码在模块里，支持 env 追加而不是完全覆盖
理由：硬编码保证开箱即用；`PAYWALL_DOMAINS_EXTRA` 允许用户加新站点（比如中文付费媒体）。未来域名列表更新只需改代码（对标 qiaomu 的做法）。

## 风险与降级

1. **Googlebot UA 被检测**：部分站点已开始查 reverse DNS（Google IP 是否真属 Google）。本方案不做 IP 伪造（只伪 UA + X-Forwarded-For），对检查 IP 的站点会失败，依赖后续 Tier 兜底。
2. **域名列表过时**：qiaomu 的列表是 2024-2025 年维护的。如有站点失效，加 WARNING 日志后用 Jina 兜底即可。
3. **curl_cffi 缺失**：已有降级到 requests 路径（见 `http_client.py:29-36`），paywall 无需额外处理。
